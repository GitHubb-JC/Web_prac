## Framework 이해하기

- 누군가 만들어 놓은 코드를 재사용 하는 것은 이미 익숙한 개발문화
- 이미 전 세계의 수많은 개발자들이 이미 해둔 것들이 있다. 그 과정속에서 자주 사용되는 부분들은 재사용이 편하게 이미 만들어져 있다.
- 위와 같은 코드들을 모아 놓은 것, 개발에 필요한 기능들을 미리 구현해서 모아 놓은 것을 'Framework' 라고 한다.
- Frame(뼈대, 틀) + Work(일)
  - 일정한 뼈대, 틀을 가지고 있다.
  - 제공하는 뼈대 , 규약을 가지고 무언가를 만들 수 있다.
- '소프트웨어 프레임 워크' 는 복잡한 문제를 해결하거나 서술하는 데 사용되는 기본 개념 구조
- 프레임워크를 잘 사용하면 웹 서비스 개발에 있어서  모든 것들을 하나부터  열까지 직접 개발할 필요 없이, 내가 만들고자 하는 본질(로직)에 집중해 개발할 수 있음
- 소프트웨어의 생산성과 품질을 높임

# 클라이언트와 서버

## 클라이언드 - 서버 구조

- 오늘날 우리가 사용하는 대부분의 웹 서비스가 클라이언트 - 서버 구조를 기반으로 동작한다

- 클라이언트와 서버 역시 하나의 컴퓨터들이며 이들의 아래와 같이 상호작용한다.

  <img src="Django 시작하기.assets/image-20220921161715368.png" alt="image-20220921161715368" style="zoom:71%;" align='left'/>

- 클라이언트
  - 웹 사용자의 인터넷에 연결된 장치 (컴, 폰)
  - 크롭 또는 파이어폭스와 같은 웹브라우저
  - 서비스를 **'요청하는'** 주체
- 서버
  - 웹 페이지, 사이트 또는 앱을 저장하는 컴퓨터
  - 클라이언트가 웹 페이지에 접근하려고 하면 서버에서 클라이언트 컴으로 웹 페이지 데이터를 보내줘 사용자의 웹 브라우저에 표시됨
  - 요청에 대해 서비스를 **'응답하는'** 주체
- 상호작용 예시 (예를 들어, google 홈페이지에 접속한다.)
  - 인터넷이 연결된 세계 어딘가에 있는 구글 컴에게 'Google 홈페이지.html' 파일을 달라고 요청
  - 구글 컴은 우리의 요청을 받고 'Google 홈페이지.html' 파일을 인터넷을 통해 전달해줌
  - 그렇게 전달받은 파일을 웹 브라우저가 우리가 볼 수 있도록 해석해준다.
- 파일을 요청한 컴, 웹 브라우저를 **'클라이언트'** 라고 하고, 파일을 넘겨주는 컴을 **'서버'** 라고 한다.
- 어떤 자원 (resource)를 달라고 요청(request)하는 쪽을 클라이언트라고 하고 자원을 제공해주는 쪽을 서버(server)라고 한다



## 정리

- 우리가 사용하는 웹은 클라이언트 - 서버 구조로 이루어져 있다.
- 앞으로 배울 것이 위의 구조를 만드는 방법
- Django는 서버를 구현하는 웹 프레임워크





# Web browser 와 Web page

## 웹 브라우저란?

- 웹에서 페이지를 찾아 보여주고, 사용자가 하이퍼링크를 통해 다른 페이지로 이동할 수 있도록 하는 프로그램
- 웹 페이지 파일을 우리가 보는 화면으로 바꿔주는(렌더링, rendering) 프로그램



## 웹 브라우저 예시

- 우리가 보고 있는 웹 페이지가 사살 HTML문서 파일의 해석본
- 웹 페이지 파일 (HTML)을 받으면 웹 브라우저가 우리가 보는 화면처럼 바꿔준다.



## 웹 페이지란?

- 웹에 있는 문서
  - 우리가 보는 화면 각각 한장 한장이 웹 페이지
- 웹 페이지 종류
  - 정적 웹 페이지
  - 동적 웹 페이지



## 정적 웹 페이지

- Static Web page
- 있는 그대로를 제공하는 것(served ad - is)을 의미
- 우리가 지금까지 작성한 웹 페이지이며 한 번 작성된  HTML 파일의 내용이 변하지 않고 모든 사용자에게 동일한 모습으로 전달되는 것
  - 서버에 미리 저장된 HTML 파일 그대로 전달된 웹 페이지
  - 같은 상황에서 모든 사용자에게 동일한 정보를 표시



## 동적 웹 페이지

- Dynamic Web page
- 사용자의 요청에 따라 웹 페이지에 추가적인 수정이 되어 클라이언트에게 전달되는 웹 페이지
- 웹 페이지의 내용을 바꿔주는 주체 == 서버
  - 서버에서 동작하고 있는 프로그램이 웹 페이지를 변경해줌
  - 이렇게 사용자의 요청을 받아서 적절한 응답을 만들어주는 프로그램을 쉽게 만들 수 있게 도와주는 프레임워크 중 하나가 바로 Django
- 다양한 서버 사이드 프로그래밍 언어 ( python, java, c++ 등 ) 사용가능
- 파일을 처리하고 데이터베이스와의 상호작용이 이루어짐
- 이 중에서 Python을 이용해서 개발할 수 있는 프레임워크 Django
